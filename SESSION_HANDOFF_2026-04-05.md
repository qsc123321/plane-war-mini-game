## 一、用户目标与已定方向
用户当前要做的是一个“先好玩、先纯前端、先不做复杂用户系统”的 HTML 飞机大战网页。

这轮会话里已经定下来的产品方向如下：

- 核心优先级：`手感和爽感`
- 战斗方向：`高压躲弹 + 击杀成就感`
- 早期单局思路：`2到3分钟短局`
- 后续扩展方向：改成 `每关打完进入下一关的闯关模式`
- 关卡数：`先做10关`
- Boss 分布：`第 3 / 5 / 10 关是 Boss 关`
- 单关时长：`90 到 120 秒`
- 通关流程：`通关后回到选关页，由玩家点下一关`
- 失败流程：`先弹出失败结算页，可重试或返回选关`
- 入口交互：`真按钮 + 全中文 + 鼠标可点`
- 闪屏调整：`普通击杀不闪，只保留近身击落 / Boss 击破闪屏`
- 声音方向：先做 `纯前端程序化音效`，不先上重资源 BGM

## 二、这次会话已经落地的内容

### 1. 早一阶段已经做过的战斗爽感改造
在战役化之前，已经把原项目从“简单自动射击原型”升级成偏高压躲弹的版本，包含：

- `Overdrive` 爆发槽
  - 擦弹和击杀可以充能
  - 空格主动开爆发
  - 爆发期间射速和火力加强
- 敌人开始主动打弹幕，不再只是刷血包和随机站桩敌人
- 连杀、近身击落、擦弹这些都能带来更强得分和反馈
- HUD、开始页、结算页改成中文
- 开始页和结算页变成中央主按钮，可鼠标点击
- 普通敌机击杀闪屏已被削掉，只保留近身击落和 Boss 击破

### 2. 本轮已经做完的“战役化 + 声音”改造
当前代码已经不是“打开直接一局”的结构，而是：

- `10关战役选关页`
  - 打开后先进选关页
  - 已解锁关卡可以点击进入
  - 关卡卡片会显示：关卡编号、关卡名、是否 Boss 关、最高分、评级/状态
- `关卡过场`
  - 点关卡后先显示关卡标题过场，再进入战斗
- `关卡战斗`
  - 普通关：撑过关卡时长即可过关
  - Boss 关：中段刷 Boss，击败 Boss 才算过关
- `结算分流`
  - 失败页：`重新挑战` / `返回选关`
  - 通关页：`返回选关`
- `本地战役进度`
  - 用 `localStorage` 保存已解锁关卡、本关最高分、通关状态、最高评级
- `基础程序化音效`
  - 使用 `Web Audio API`
  - 不依赖外部音频文件
  - 已接入的声音事件包括：
    - 按钮点击
    - 玩家射击
    - 擦弹
    - 普通击落
    - 近身击落
    - 拾取道具
    - 护盾受击
    - 玩家受伤
    - 爆发启动
    - Boss 警报
    - Boss 击破
    - 通关
    - 失败

## 八、我认为新窗口最值得优先做的事
如果新窗口要继续往前推进，我建议按下面优先级：

### P0：先做真实试玩和数值校准
重点先玩：
- 第 1 关是否太弱
- 第 2 关到第 3 关难度爬升是否顺
- 第 3 关 Boss 会不会拖太久
- 第 5 关和第 10 关会不会过难或过乱
- 失败后“重试 / 返回选关”是否顺手

### P0：调音效密度
因为现在声音是新加的，最容易出现两个问题：
- 射击音太密、太刺耳
- 擦弹 / 击落 / 受伤混在一起，信息过载

新窗口很适合专门做一次“音效节奏调优”。

### P1：补一点选关页视觉完成度
当前选关页已经能用，但还可以继续增强：
- 关卡卡片的 Boss 关识别更强
- 已通关 / 未通关 / 未解锁三种状态的视觉差异更明确
- 选中态更醒目
- 返回选关后的焦点默认落到刚解锁或刚打过的那一关

### P1：正式重新部署
因为当前代码和旧线上链接不同步，下一窗口如果要给用户试玩，应该重新打包并部署一次。
