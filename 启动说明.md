# 飞机大战小游戏启动说明

## 一、项目位置
项目目录：`E:/飞机大战/飞机大战小游戏`

请注意：**必须进入这个目录再执行命令**，不要在上一级目录执行。

---

## 二、运行前准备
这个项目是 **Vite + TypeScript** 网页项目，需要先安装：

- Node.js
- npm

你可以先在终端里检查是否已安装：

```bash
node -v
npm -v
```

如果能看到版本号，说明环境已经安装好。

---

## 三、正确启动方式
### 1. 打开终端
可以使用：
- VSCode 终端
- PowerShell
- Windows Terminal
- cmd

### 2. 进入项目目录
```bash
cd "E:/飞机大战/飞机大战小游戏"
```

### 3. 安装依赖
第一次启动时先执行：

```bash
npm install
```

如果之前已经安装过，一般不用重复执行。

### 4. 启动开发服务器
```bash
npm run dev
```

启动成功后，终端里会出现类似这样的地址：

```text
http://localhost:5173/
```

### 5. 打开浏览器访问
把终端显示的地址复制到浏览器打开。

通常是：

```text
http://localhost:5173/
```

---

## 四、不要这样启动
### 错误方式：直接双击 `index.html`
**不要直接双击打开 `index.html`。**

原因：
- 这个项目不是普通静态网页
- 它依赖 Vite 开发服务器来处理 `src/main.ts`
- 如果直接用 `file://` 打开，会出现：
  - CORS 报错
  - `Failed to load resource`
  - 页面白屏

也就是说：

**必须使用 `npm run dev` 启动后，再通过浏览器访问本地地址。**

---

## 五、关闭项目
在终端里按：

```text
Ctrl + C
```

即可停止开发服务器。

---

## 六、常见问题
### 1. 提示 `npm` 不是内部或外部命令
说明没有安装 Node.js，或者环境变量没有配置好。

解决办法：
- 安装 Node.js
- 安装后重新打开终端再试

### 2. 提示找不到 `vite`
通常是因为还没有安装依赖。

执行：

```bash
npm install
```

然后再执行：

```bash
npm run dev
```

### 3. 页面还是打不开
请检查：
- 是否在 `E:/飞机大战/飞机大战小游戏` 目录执行命令
- 是否先执行过 `npm install`
- 是否执行的是 `npm run dev`
- 浏览器打开的是否是终端输出的地址，而不是本地 `index.html`

### 4. 打开后是白屏
先按下面顺序检查：
1. 有没有直接双击 `index.html`
2. 是否打开的是 `http://localhost:5173/`
3. 终端有没有报错
4. 浏览器控制台有没有报错
